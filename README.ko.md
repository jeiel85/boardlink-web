## ▶ 지금 플레이

**라이브:** https://boardlink.jeiel85.workers.dev

BoardLink는 이미 배포되어 바로 플레이할 수 있습니다:

- **컴퓨터와 대결** (같은 기기, 네트워크 불필요) — 오목·체스·장기(쉬움/보통/어려움
  난이도), 빙고(자동 상대), 그리고 실시간 버블시즈 아레나(반응형 봇).
- **온라인 플레이** — 방을 만들어 6자리 코드를 공유하고, 오목·체스·장기를 WebSocket으로
  실시간 대전. 매치 중 재접속하면 보드가 복원됩니다.
- **익명** — 첫 방문 시 기기 신원이 생성됩니다. 계정·이메일·로그인 없음. PWA로 설치 가능.

## 제품 개요

BoardLink는 휴대폰·태블릿·컴퓨터에서 링크를 열기만 하면 곧바로 짧은 멀티플레이어 게임을 할 수 있게 합니다.

핵심 진입 경로:

1. **주변에서 찾기** — 같은 인터넷 연결을 공유하는 것으로 보이는 사용자를 발견합니다.
2. **온라인 친구** — 친구 코드로 익명 기기 프로필을 찾습니다.
3. **빠른 입장** — 방 코드 입력, QR 스캔, 초대 링크 열기.
4. **같은 기기 플레이** — 네트워크 없이 로컬에서 플레이.
5. **PWA 설치** — 선택적 앱 형태 설치.

초기 릴리스는 계정, 이메일, 전화번호, 광고 SDK, 분석 SDK, 소셜 로그인, 공개 채팅을 요구하지 않습니다.

## 게임

5개 런칭 게임 모두 공유·서버 권위 게임 SDK 위에 결정론적 모듈로 구현되어 있습니다(모든 매치는 이벤트 로그로 재생 가능).

| 게임          | 컴퓨터 대결    | 온라인 | 비고                                          |
| ------------- | -------------- | ------ | --------------------------------------------- |
| 오목 (Gomoku) | ✅ (3단계)     | ✅     | 15×15, 5목                                    |
| 체스 (Chess)  | ✅ (3단계)     | ✅     | 전 규칙: 캐슬링·앙파상·프로모션               |
| 장기 (Janggi) | ✅ (3단계)     | ✅     | 한국 장기 — 궁성 이동·포·외통                 |
| 빙고 (Bingo)  | ✅ (자동 상대) | —      | 턴제 뽑기 / 마킹 / 빙고 선언                  |
| 버블시즈      | ✅ (3단계)     | —      | 원작 비대칭 실시간 아레나; 온라인은 다음 단계 |

컴퓨터 상대는 게임별 평가함수를 쓰는 제네릭 알파-베타 엔진입니다(난이도 = 탐색 깊이). 빙고는 그리디 자동 플레이어를 씁니다.

이후: 팀·파티 모드, WebRTC 직결 전송, 관전, 토너먼트, 실험적 오프라인 QR 페어링.

## 권장 스택

- React 19
- TypeScript
- Vite
- React Router
- Cloudflare Vite 플러그인
- Cloudflare Workers
- WebSocket Hibernation을 사용하는 Cloudflare Durable Objects
- 익명 공개 친구코드 디렉터리용 D1
- 로컬 프로필·친구·설정·로컬 기록용 IndexedDB
- 런타임 스키마용 Zod
- Vitest 및 `@cloudflare/vitest-pool-workers`
- 종단 간 브라우저 테스트용 Playwright
- `vite-plugin-pwa` 또는 직접 관리하는 Workbox 서비스 워커

## 타협 불가 제약

- 일회성 MVP가 아니라 프로덕션 지향 설계.
- 서버 권위 멀티플레이어 상태.
- 규칙·점수·타이머·신원 소유·방 권한에 대해 어떤 클라이언트도 신뢰하지 않음.
- 주변 발견은 근사치이며 정확한 물리적 근접으로 설명해서는 안 됨.
- 지원되지 않는 인앱 브라우저에서는 실시간 플레이 차단.
- PWA 업데이트는 매치 도중에 절대 활성화되지 않음.
- 모든 프로토콜 메시지는 버전 관리·스키마 검증됨.
- 모든 게임 리듀서는 결정론적.
- 모든 공개 식별자는 교체 가능하며, 내부 식별자는 직접 노출되지 않음.
- 방문자 원본 IP는 애플리케이션 코드에 의해 저장되지 않음.
- 초기 릴리스에 공개 자유 채팅 없음.
- 초기 릴리스에 랭크 모드나 상품 없음.
- 서드파티 분석·광고·추적 스크립트 없음.

## 출시 준비 정의

다음을 모두 만족해야 릴리스 후보가 출시 준비 완료입니다:

- 실제 기기 두 대가 발견·초대·입장·플레이·연결 해제·재접속할 수 있음.
- Android Chrome, 삼성 인터넷, iOS Safari, 설치된 iOS PWA, Windows Chrome/Edge, macOS Safari/Chrome 지원 매트릭스 통과.
- 카카오톡 등 테스트된 인앱 브라우저에서 안전한 외부 브라우저 전환 표시.
- 버블시즈가 결정론적 타이밍·결과 테스트를 통과.
- 룸 복원, 오래된 클라이언트 거부, 멀티탭 잠금, PWA 업데이트 연기, 레이트 리밋이 테스트됨.
- 개인정보 고지, 서비스 약관, 삭제/초기화 컨트롤, 연락 채널이 게시됨.
- 운영 대시보드와 롤백 절차가 문서화됨.

## 개발 및 배포

### 로컬 개발

1. 의존성 설치:
   ```bash
   pnpm install
   ```
2. 로컬 개발 서버 시작 (Vite 클라이언트 빌드 + Wrangler dev 실행):
   ```bash
   pnpm dev
   ```

### 품질 및 테스트

커밋 전에 모든 품질 검사를 실행하세요:

```bash
# 코드 포맷 검사
pnpm format:check
pnpm format  # 포맷 수정 적용

# 코드 린트
pnpm lint

# TypeScript 컴파일 검사
pnpm typecheck

# 패키지 단위 테스트
pnpm test

# Workers 런타임 통합 테스트
pnpm test:workers

# 종단 간 스모크 테스트 (Playwright)
pnpm test:e2e
```

### 프로덕션 빌드

공유 패키지와 플랫폼 클라이언트 에셋/워커 서버를 빌드:

```bash
pnpm build
```

### 배포

Cloudflare에 배포하려면:

1. Wrangler CLI로 Cloudflare 계정에 로그인:
   ```bash
   pnpm exec wrangler login
   ```
2. 플랫폼 애플리케이션 배포:
   ```bash
   pnpm --filter @boardlink/platform deploy
   ```

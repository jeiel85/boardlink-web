# Game Expansion Strategy

## Goal

Expand BoardLink from a simple browser board-game set into a **multi-situation multiplayer platform**.

That means the catalog should support:

- 2-player strategic play
- short reaction games
- family-friendly games
- small-group gatherings
- church / school / event usage
- quick 30-second games
- longer 10-minute games

## Portfolio structure

A healthy catalog should not be all one type.

Recommended balance:

```text
Strategy
├─ Gomoku
├─ Four in a Row
├─ Reversi
├─ Chess
└─ Janggi

Family / Familiar
├─ Bingo
├─ Number Baseball
└─ Yutnori

Party / Utility
├─ Quiz Buzzer
├─ Rock-Paper-Scissors Tournament
├─ Secret Role
└─ Majority Prediction

BoardLink Originals
├─ Bubble Siege
├─ Tap Territory
├─ Color Catcher
├─ Sync Bomb
├─ Memory Raid
└─ Path Block
```

## Why this matters

If the service contains only classic strategy games:

- it becomes narrow,
- large-group usage is weak,
- repeatability depends on a few users,
- the product looks replaceable.

If the service also contains gathering-friendly games:

- the same platform works for family play,
- school/church/community events become possible,
- room and invitation systems become more valuable,
- the platform gets more natural re-entry points.

## Selection criteria

Every added game should be scored using these factors:

1. **Understanding speed**
2. **Input simplicity**
3. **Transport simplicity**
4. **Browser fit**
5. **Replayability**
6. **Social value**
7. **Platform reuse**
